# 하네스톤 2026 트랙2 제출물 — pptx-to-web

## 5. 프로젝트 소개 및 구현 내용

### pptx-to-web — 멈춰있는 발표자료를 "항상 최신인 라이브 웹 데모"로 바꾸는 Copilot Skill

> PPT는 만드는 순간 낡습니다. pptx-to-web은 발표자료를 웹으로 바꿔, **GitHub Copilot에게 한마디만 하면 최신 정보로 갱신되고, 고객에게는 URL 하나로 즉시 전달**됩니다. 더 이상 파일 첨부·버전 관리·재배포가 필요 없습니다.

### 해결하고자 한 문제
PowerPoint 발표 자료를 웹 버전으로 만들어, 최신 정보를 GitHub Copilot 기반으로 쉽게 업데이트하고 고객에게 즉시 전달하는 것이 목적입니다. 매번 PPT를 수동 수정·재배포하는 번거로움 없이, 에이전트가 최신 내용을 반영해 라이브 사이트를 갱신합니다.

### 이 스킬이 주는 이점
- **항상 최신**: Azure/Foundry 공식 "What's new"를 자동 반영 — 수동 수정 불필요
- **즉시 전달**: 파일 대신 URL — 고객은 클릭만, 누구나 같은 최신본
- **고품질 자동화**: 에이전트가 슬라이드를 직접 보고 다이어그램으로 재해석 → 빈 장표 0
- **유지보수 제로**: "이 수치 바꿔줘"만으로 사이트가 갱신·재배포

### Harness Asset 유형 / 목적
- 유형: **GitHub Copilot Skill** (`.github/skills/pptx-to-web/`)
- 목적: PPTX → 다이어그램 중심 웹 덱 자동 생성, 최신 정보 반영, 고객에게 URL로 즉시 전달

### 주요 기능
1. 캡처: 모든 장표를 PNG로 렌더(capture.py)
2. AI 재해석: 에이전트가 PNG를 보고 발표자 노트 → 박스·도넛·허브·플로우 SVG로 재구성
3. 최신화: Azure/Foundry 공식 "What's new" 자동 수집·삽입 (GHCP로 한 줄 요청이면 갱신)
4. 빌드: 좌측 목차 + 1화면 랜드스케이프 스냅 + 6색 테마 + 발표 노트
5. 검증: headless 렌더로 빈 슬라이드 0 자동 보장
6. 배포: GitHub Pages — 고객에게 URL만 공유

### 사용 방법
스킬 하나만 호출 → capture → content.json 작성(templates 참고) → 빌드 → push. 이후 내용 변경은 Copilot에게 요청만 하면 사이트가 자동 갱신됩니다.

### 활용 대상
최신 정보를 자주 반영해 고객에게 발표·공유해야 하는 SE·세일즈·교육자.
